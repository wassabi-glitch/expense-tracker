# Grill Me

Interview the user deeply before proposing or implementing a solution.

Rules:

1. Ask one focused question at a time.
2. Walk the decision tree branch by branch.
3. For each question, include a recommended answer or senior-level leaning.
4. If the answer can be discovered by inspecting the codebase, inspect the codebase instead of asking.
5. Do not rush to implementation until the important assumptions, constraints, tradeoffs, and consequences are clear.
6. Summarize decisions when the branch is resolved.
7. If the user becomes overwhelmed, compress the discussion into a short decision map.
